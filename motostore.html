MotoStore Mobile Check Out & Atomic Design

Since the start of may I have been working at Vilh. Nellemann Handelsselskab. Here I have been working on UI design for their webshop motostore.dk and their website honda-mc.dk. One of the assignments I have worked on is a new design for their checkout on motostore.dk

After making a quick initial design I decided to read up on the best practices of mobile check out designs. I got some useful insights which impacted my design. Some of them I had already followed. Like using buttons for progressing as well as using a progress indicator to let the customer know that they aren't far away from making their purchase.

One of the insights I got was to eliminate distractions and keeping the design simple. This meant that I reduced the amount of visual clutter by removing color from the button "Ordredetaljer" and replacing it with a stroke aswell as removing colorful elements around titles and subtitles. By doing so it also increased the focus on progression buttons so it was actually two birds with one stone.

I was reminded of the power of social logins which made me incorporate facebook and google login butttons. I also learned that using floating labels with in form field is a great for the user experience as it will help remind the user what type of information they are supposed to type in.

Atomic Design

Whilst doing this design i stumbled upon Atomic Design. It's a design philosophy that separates design elements into building blocks of increasing complexity. I tried applying this philosophy to my work space in figma. The most basic building blocks are Atoms. They consist of elements such as buttons, labels, form fields etc.

Combining multiple of these atoms I created molecules which I put into a separate figma page.

Further combining molecules and atoms created organisms.

These organisms I applied as building blocks to the final design. Some of them were had more complex interaction like dropdowns witin dropdowns. I learned a lot about auto layout and nested components when I had to make the organism for choosing the delivery method.

I definitely intend to use this design philosophy in future work as it makes doing changes to basic elements across multiple designs really fast as well as providing structure and overview of design elements. I can see how the benefits gets exponentially bigger as projects increase in size.
